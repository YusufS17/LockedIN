  <action>
    Create LockedIN/Session/SessionCoordinator.swift as an @Observable final class (no ObservableObject/Combine per CLAUDE.md).

    Properties required (all @Observable — no @Published):
    - contract: CommitmentContract (set at init)
    - sessionDeadline: Date (set at startSession; nil = not started)
    - shieldIsActive: Bool = false  — drives .fullScreenCover in SessionView
    - thresholdExceeded: Bool = false  — flips to true when distractionCount crosses contract threshold; drives ForfeitWarningView inside the cover
    - distractionCount: Int = 0  — increments on each recorded distraction
    - permittedDistractionEvents: Int  — read from contract; expose as computed let for display

    permittedDistractionEvents is derived from contract.maxDistractionSeconds divided by a per-event budget constant (use 300 seconds = 5 min per event as a named constant `distractionEventBudgetSeconds = 300`), rounded down, minimum 1. Expose this Int so the overlay can render "X of Y permitted distraction events".

    Methods:
    - startSession() — sets sessionDeadline = Date().addingTimeInterval(Double(contract.durationSeconds)), launches a Task that calls focusAdapter.startMonitoring(participantID: userID) and iterates the AsyncStream; for each .distractionStarted event with the user's participantID, call handleDistractionEvent(). Store the Task in a private var monitoringTask: Task<Void,Never>? for cancellation.
    - handleDistractionEvent() — increments distractionCount; sets shieldIsActive = true; if distractionCount > permittedDistractionEvents sets thresholdExceeded = true.
    - returnToRoom() — sets shieldIsActive = false (does not record a distraction; contract unchanged).
    - usePlannedBreak() — sets shieldIsActive = false (break path; full break logic deferred to Phase 6 rules; just dismiss shield for now).
    - recordDistraction() — already counted in handleDistractionEvent(); just sets shieldIsActive = false.
    - endSessionEarly() — cancels monitoringTask, sets shieldIsActive = false; posts a Notification named "SessionEndedEarly" so SessionView or its parent can navigate away. Use NotificationCenter.default.post(name: .sessionEndedEarly, object: nil); declare the Notification.Name extension in this file.
    - confirmForfeit() — same as endSessionEarly() but can be called from the hold-confirm path on ForfeitWarningView.

    timeRemaining computed property: returns max(0, sessionDeadline?.timeIntervalSinceNow ?? 0) as TimeInterval.

    The userID is injected at init as a UUID parameter (default UUID() for preview use).

    Import only Foundation and Observation. Do NOT import Combine or SwiftData.

    Place file at LockedIN/Session/SessionCoordinator.swift; create the Session/ group if it does not exist.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → shield choices | User taps buttons on ShieldOverlayView; must only trigger coordinator methods, never bypass shield |
| Hold gesture → forfeit confirm | LongPressGesture is the sole path to confirmForfeit(); accidental taps must not trigger it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | ShieldOverlayView dismiss | mitigate | .interactiveDismissDisabled(true) on presented cover — swipe-to-dismiss is structurally disabled (SHD-01) |
| T-05-02 | Deception | Button labels | mitigate | All three choices use exact spec labels; "End session early (forfeit £X)" must name the amount explicitly (SHD-03, SHD-05, SAFE ethical line) |
| T-05-03 | Tampering | Hold-to-confirm bypass | mitigate | LongPressGesture(minimumDuration: 1.5) — single tap cannot confirm forfeit; deliberate hold required (SHD-04) |
| T-05-04 | Information Disclosure | App name in overlay | accept | ShieldOverlayView never displays which app was blocked — FocusEvent carries no app identity (SAFE-02); only "a blocked app" language used |
| T-05-05 | Denial of Service | endSessionEarly path hidden | mitigate | Forfeit exit rendered unconditionally in ShieldOverlayView body outside the thresholdExceeded branch — always visible on both states (SHD-05) |
</threat_model>
