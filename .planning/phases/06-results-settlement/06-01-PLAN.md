    Create LockedIN/Services/SettlementCoordinator.swift.

    Define a struct SettlementCoordinator (not a class — stateless coordinator, no @Observable). Give it a single static async method:

      static func settle(
          participants: [Participant],
          holdRefs: [UUID: HoldReference],
          metrics: [UUID: ParticipantMetrics],
          contract: CommitmentContract,
          supportiveMode: Bool,
          commitmentService: any CommitmentService
      ) async -> SessionResult

    ParticipantMetrics is the metrics type Phase 4 produces. If it does not exist yet, declare a local typealias or a minimal placeholder struct with fields focusedSeconds (Int), distractionCount (Int), overrideAttempts (Int), leftEarly (Bool), focusPercent (Int) — clearly comment "replace with Phase 4 type once available".

    Inside the method:
    1. For each participant, look up their holdRef. If nil (no stake held), set state to notRequired, verdict to passed, stakeOutcomePence 0.
    2. If holdRef found, call commitmentService.settle(holdRef:verdict:) in a do/catch. Determine verdict from metrics: if distractionCount > contract.maxDistractionEvents OR leftEarly == true → .failed; else → .passed. On success, state = .returned(record:) or .forfeited(record:). On catch, state = .settlementError(message: error.localizedDescription).
    3. Assert money conservation per participant inside the do block: record.returnedMinorUnits + record.forfeitedMinorUnits == holdRef.amountMinorUnits. Use assert() (prototype only).
    4. Assign stakeOutcomePence: returned → +record.returnedMinorUnits, forfeited → -record.forfeitedMinorUnits, error/notRequired → 0. All Int, no floats.
    5. Assign titles after all settlement is done:
       - lockedInChampion: the passer with highest focusPercent (ties: alphabetical displayName)
       - biggestCulprit: the failer with highest distractionCount — if supportiveMode is true, use honourableParticipant instead
       - firstToFold: the failer who failed earliest (lowest focused seconds among failers who are not biggestCulprit, or secondarily alphabetical); skip if supportiveMode true
       - everyone else: honourableParticipant
    6. Build RoomTotals: focusedMinutes = sum of focusedSeconds across all participants divided by 60 (integer division); totalReturnedPence = sum of +stakeOutcomePence (passers); totalForfeitedPence = abs sum of -stakeOutcomePence (failers); successfulCount = passer count; participantCount = participants.count; roomFocusPercent = sum of focusPercent / participantCount (integer division, guard against division by zero → 0).
    7. Sort results array: lockedInChampion first, honourableParticipant passers next (alphabetical), firstToFold before biggestCulprit among failers, biggestCulprit last.
    8. Return SessionResult(roomTotals:results:isTestMode:forfeitDestination:) — isTestMode pulled from the first successful SettlementRecord.isTestMode, defaulting to true. forfeitDestination from ForfeitConfig.destination.

    Do not import SwiftUI. Foundation only. No UI dependencies.
