| SessionResult → display | All money values from SettlementRecord (Int pence) pass through MoneyLabel before display |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | MoneyLabel TEST marker | mitigate | MoneyLabel structural design: TEST marker cannot be bypassed — it is part of the component, not conditional in the caller |
| T-06-06 | Tampering | stakeOutcomePence sign in TitleRevealView | mitigate | abs(stakeOutcomePence) passed to MoneyLabel; sign used only for "Returned"/"Forfeited" label — no arithmetic in views |
| T-06-07 | Spoofing | supportiveMode suppression of negative titles | accept | View reads result.title.displayName directly — coordinator already applied suppression; no second conditional in view required |
| T-06-08 | Denial of Service | settleSession called multiple times | mitigate | AppStore.settleSession is idempotent (no-op if sessionResult already set); .task runs once per view appearance |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan — SwiftUI views only, no new dependencies |
