<threat_model>
## Trust Boundaries

No network, no backend, no real money. The integrity boundaries are the **settlement state machine** (money must be conserved across transitions) and the **focus-event surface** (must never expose private app/usage data).

| Boundary | Description |
|----------|-------------|
| held stake -> settled outcome | Settlement must conserve money exactly; no float, no partial loss/creation |
| FocusEvent emission | Aggregate-only signal; must never carry app names, URLs, messages, contacts, notifications |
| protocol boundary (service seam) | UI/session must reach payment/focus only via protocols, never concrete mocks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering (money conservation) | MockCommitmentService.settle | mitigate | `settle()` uses integer Pence only; returned + forfeited == held amount asserted per verdict; no Double/Float — a settlement cannot silently lose or create money (FND-01, FND-04) |
| T-01-06 | Information disclosure (private data) | FocusEvent / MockFocusControlAdapter | mitigate | `FocusEvent` cases carry only participantID + Date; no app names/URLs/message/notification fields ever (SAFE-02) — grep confirms no such fields in Models |
| T-01-07 | Tampering (invalid state transition) | ParticipantSettlementState | mitigate | Exactly the 8 canonical states modelled as a typed enum (not String) so no ad-hoc/typo state can exist (FND-04, D-11) |

| T-01-09 | Spoofing (economy conflation) | SettlementRecord, money fields | mitigate | This phase has only commitment £ (no coins yet); SettlementRecord money is all `MinorUnits`; coins are a Phase-6 concern kept structurally separate (SAFE-04) |
