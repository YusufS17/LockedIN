## Phase Goal

**As a** LockedIN developer building the demo spine, **I want to** a money-correct, safety-first SwiftUI substrate that builds and renders a `£` amount through one TEST-marked component, **so that** every later phase consumes the same money type, design tokens, and structurally-enforced TEST MODE label instead of reinventing them per screen.

## Acceptance Criteria
- [ ] `xcodebuild build` for the iOS 17 simulator scheme exits 0
- [ ] Root screen renders a seeded `Pence` value as `£X.XX` with the TEST marker visible alongside it
- [ ] No money figure can be rendered without the TEST marker (only `MoneyLabel` formats `£`)
- [ ] All 4 feature flags exist with locked values; `ENABLE_REAL_MONEY_STAKES=false` is read at the render boundary
- [ ] `AppStore` injected via `.environment()` at the `WindowGroup` root

<threat_model>
## Trust Boundaries

This phase has NO network, NO backend, NO real money. The trust boundary is the **money-rendering boundary**: any code path that converts a stored `Pence` into a user-visible `£` figure. Integrity/safety threats (not network threats) apply.

| Boundary | Description |
|----------|-------------|
| stored Pence -> rendered £ figure | The only place money becomes user-visible; SAFE-01 requires the TEST marker here |
| feature-flag declaration -> behaviour | A flag declared but not read at its boundary is a silent integrity gap (FND-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure / Repudiation | Any view rendering `£` | mitigate | `MoneyLabel` is the ONLY sanctioned £ renderer and ALWAYS emits the TEST marker (D-02); `grep` gate confirms `formatPence` is called only inside MoneyLabel — rendering money without the marker is structurally impossible (SAFE-01) |
| T-01-02 | Tampering (data integrity) | Money fields | mitigate | `typealias Pence = Int`; `grep -rn ": Double\|: Float\|: Decimal" LockedIN/` over money fields returns nothing — float rounding cannot corrupt a £ figure (FND-01, SAFE-03) |
| T-01-03 | Elevation of privilege (flag bypass) | FeatureFlags | mitigate | `ENABLE_REAL_MONEY_STAKES=false` is READ at the MoneyLabel render boundary so the marker is always on; flag is honoured not merely declared (FND-05, SAFE-02) |
| T-01-04 | Spoofing (deceptive UI / dark pattern) | Theme tokens, MoneyLabel | mitigate | Design tokens and the TEST marker are honest and legible (D-05); no hidden/disguised controls; full "TEST MODE — NO REAL MONEY WILL MOVE" statement always reachable (D-03) |
| T-01-SC | Tampering (supply chain) | SPM / dependencies | mitigate | Zero third-party dependencies; `grep` gate confirms no `packageReferences` in project.pbxproj — no install step, no untrusted package surface (the package-legitimacy gate is N/A: no installs) |
</threat_model>
