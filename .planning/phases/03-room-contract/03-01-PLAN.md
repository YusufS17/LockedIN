  <name>Task 1: RoomStore — preset contract, settlement map, and bot choreography (per D-16, D-17, D-18)</name>
  <files>LockedIN/Stores/RoomStore.swift</files>
  <action>
    Create `LockedIN/Stores/RoomStore.swift` as a new directory and file. The `Stores/` group is auto-included in the Xcode project because the project uses a synchronized root group — no pbxproj edit needed.

    Declare `@Observable final class RoomStore`. Inject `AppStore` in the initialiser and store it as a private property.

    Define the preset contract as a static let or lazy property (force-unwrap the failable init — the hardcoded values are valid): roomName "£5 Serious Lock-In", durationSeconds 1500 (25 min per D-16), stakeMinorUnits 500 (£5.00), maxDistractionSeconds 180 (3 × 60s to represent ≤3 distraction events), maxBreaks 1, currencyCode "GBP".

    Observable properties:
    - `var contract: CommitmentContract` — starts unfrozen; set to `.frozen()` copy after Sam accepts
    - `var participants: [Participant]` — ordered array: the user participant (isBot: false, displayName: "You") then Maya, Leo, Sam (isBot: true). Initialise in init with hardcoded UUIDs so they remain stable within a session.
    - `var settlementStates: [UUID: ParticipantSettlementState]` — keyed by participant.id; all start as `.awaitingAuthorisation`
    - `var totalStakedPence: Pence` — computed from count of `.held` states × stakeMinorUnits (500)
    - `var isFrozen: Bool` — mirrors contract.isFrozen
    - `var userHasAccepted: Bool = false`

    Bot participant IDs: declare three `static let` UUID constants (maya, leo, sam) using fixed UUID string literals so IDs are deterministic across previews.

    User participant ID: expose as `let userParticipantID: UUID = UUID()` created once in init.

    Method `func acceptAndStake(participantID: UUID) async`: call `appStore.commitmentService.authoriseHold(participantID: participantID, amountMinorUnits: 500, contract: contract)`, on success set `settlementStates[participantID] = .held(ref: holdRef)`. Catch and set `.settlementError(message:)` on failure.

    Method `func startBotSequence() async`: after user accept, fire three Tasks in sequence (not in parallel — deliberate cadence per D-17):
    1. `try? await Task.sleep(for: .seconds(2))` then call `acceptAndStake(participantID: RoomStore.mayaID)`
    2. `try? await Task.sleep(for: .seconds(2))` then call `acceptAndStake(participantID: RoomStore.leoID)`
    3. `try? await Task.sleep(for: .seconds(2))` then call `acceptAndStake(participantID: RoomStore.samID)`, then set `contract = contract.frozen()`

    Store the returned `Task` from `startBotSequence` in a private `var botTask: Task<Void,Never>?` so it can be cancelled on deinit. Cancel in `deinit`.

    Do NOT import Combine. Do NOT use @Published. Do NOT use ObservableObject.
  </action>
