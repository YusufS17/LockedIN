## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User ↔ App (display-name text input) | The display-name TextField in PayoffBeatView accepts free-text user input that is rendered back in headings and persisted. |
| App ↔ UserDefaults / @AppStorage | The first-launch flag (`hasCompletedOnboarding`) and identity are read on launch to decide routing; treated as local, non-sensitive, and possibly stale. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering / display overflow | Display-name TextField (over-long / whitespace-only input) | mitigate | `.onChange` truncates to 30 chars at the input boundary (V5); CTA guards on `trimmingCharacters(in: .whitespaces).isEmpty` so blank/space-only names cannot complete onboarding; SwiftUI `Text` renders the name as inert text (no markup/HTML execution) |
| T-02-07 | Information Disclosure | Rendering/logging the display name | mitigate | Name is shown only in the user's own headings/Home greeting; do not `print`/`os_log` the name or appearance; no network egress in this phase (SAFE-02 spirit: no private data leaves the device) |
| T-02-08 | Denial of Service (crash) | Routing on stale @AppStorage / persisted identity after schema change | mitigate | `@AppStorage` bool defaults to false (re-show onboarding rather than crash); identity load uses CharacterPersistence's `try?` fallback to `.default` (from 02-02) — no force-unwrap on the launch path |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs (CLAUDE.md forbids SPM; RESEARCH §Package Legitimacy Audit: not applicable). No supply-chain surface. |
</threat_model>
