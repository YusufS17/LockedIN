## Screen-by-Screen Interaction Contract

### Screen 1 — Concept Beat (`ConceptBeatView`)

**Layout:** Full-bleed `VStack`, background `Theme.Colour.background`, safe-area insets respected.

**Animated elements:**
- 3 lines ("Stake it." / "Lock in." / "Win it back.") appear sequentially via `withAnimation(.easeOut(duration: 0.5))` staggered with `Task.sleep` delays (0.0s / 0.6s / 1.2s from trigger). Each uses `.opacity` + `.offset(y: 20 → 0)` entrance. Line 3 uses `Theme.Colour.accent` fill; Lines 1–2 use `Theme.Colour.textPrimary`.
- Reduce Motion fallback: all three lines appear simultaneously via `.opacity` fade only (no offset), no stagger. Check `@Environment(\.accessibilityReduceMotion)` before each animation.
- Supporting body copy fades in at 1.8s (or immediately with Reduce Motion).
- CTA button fades in at 2.2s (or immediately with Reduce Motion).

**Interactions:**
- "Get started" pill button → `withAnimation(.spring(duration: 0.4, bounce: 0.2))` advances `beat` to `.create`; uses `.asymmetric` transition (slide trailing in, slide leading out).
- "Skip" text button → sets `hasCompletedOnboarding = true` in `@AppStorage`, sets `AppStore.userCharacter = CharacterAppearance.default`, navigates to `HomeView`. No confirmation required — skip is honest and immediate.

**Progress dots:** 3 dots, horizontal, bottom of screen above safe area. Active dot: 8×8 pt circle, `Theme.Colour.accent`. Inactive: 6×6 pt, `Theme.Colour.textSecondary` at 40% opacity.

### Screen 2 — Character Creator (`CharacterCreatorView`)

**Layout:** `VStack(spacing: Theme.Spacing.xl)` — heading, avatar preview centred, 4 selector rows, CTA.

**Avatar preview (live):**
- `AvatarView(appearance: localAppearance, status: .idle, size: 120)` centred horizontally.
- Updates instantly on every stepper tap (bound to local `@State var localAppearance: CharacterAppearance`).
- No animation on layer swap — instant redraw (animated transitions here distract from the selection UX).
- Preview sits on a `RoundedRectangle` card background (`Theme.Colour.surface`, `Theme.Radius.lg`) for contrast.

**Selector rows (4 rows):**
Each row is an `HStack`:
- Label: Headline font, `Theme.Colour.textSecondary`, fixed left.
- `Spacer()`
- `◀` button: 44×44 pt tap target, SF Symbol `chevron.left`, `Theme.Colour.textPrimary`.
- Value label: 80pt minimum width, Body font, `Theme.Colour.textPrimary`, centred.
- `▶` button: 44×44 pt tap target, SF Symbol `chevron.right`, `Theme.Colour.textPrimary`.

Row order (top to bottom): Skin · Hair · Outfit · Accent.

Selector options (cycle wrapping — tapping ◀ on first item wraps to last):
- Skin: Light / Medium / Dark / Deep (4 options)
- Hair style: Short / Curly / Long / Tied (4 options — style names, not colour)
- Outfit: Casual / Academic / Hoodie / Smart (4 options)
- Accent: Amber / Teal / Rose / Lavender (4 options — shown with a 16×16 colour swatch circle beside the name)

For Skin and Accent rows: show a 16×16 pt filled `Circle` swatch (skin tone colour / accent colour) to the left of the value label — non-colour-only identification (swatch + name).

**CTA:** "Looks good" pill button, full-width with `Theme.Spacing.lg` horizontal inset, `Theme.Colour.accent` background, `Theme.Colour.textOnAccent` label. Always enabled (default pre-selected; never invalid).

**Skip:** Same as Beat 1. Positioned top-right as a secondary text button.

**Progress dots:** Same treatment; 2nd dot active.

### Screen 3 — Payoff Beat (`PayoffBeatView`)

**Layout:** ZStack — `IsometricRoomView` fills the full screen behind; a foreground `VStack` overlay is anchored to the bottom third with the heading, name field, and CTA.

**Avatar entrance sequence (PhaseAnimator):**
- Triggered once `showAvatar` is set to `true` (after TextField is visible for 0.5s, or immediately if name was entered before avatar shown).
- Phases: `.hidden` → `.riseUp` → `.settle` as defined in RESEARCH.md Pattern 5.
- Reduce Motion: skip scale/offset; show avatar immediately at `.opacity` 0→1 only (no scale, no offset).
- Avatar size: 80×80 pt, positioned at the room desk centre point (see IsometricRoomView placement contract below).

**IsometricRoomView in this context:** Full-bleed, no overlaid HUD. The room IS the background. No participant slots yet (Phase 4 adds those) — avatar placement is a fixed desk centre position.

**Foreground overlay:**
- `VStack(spacing: Theme.Spacing.md)`, semi-transparent gradient `LinearGradient` from `Theme.Colour.background.opacity(0)` to `Theme.Colour.background.opacity(0.92)` (bottom-to-top), so room is still visible through the upper part of the overlay.
- Pre-name heading: "Almost there — what's your name?" (Title, textPrimary).
- `TextField("Your name", text: $displayName)` — Body font, `Theme.Colour.textPrimary`, on a `Theme.Colour.surface` rounded background (`Theme.Radius.sm`), padding `Theme.Spacing.md` inset.
- When displayName is non-empty: animate heading swap to "Meet [displayName]." with `.transition(.opacity)` cross-fade — no abrupt snap.
- CTA "Let's go": disabled (opacity 0.4) until `displayName.trimmingCharacters(in: .whitespaces).count >= 1`. Enabled: amber fill, textOnAccent label.
- Inline hint (empty submit): "Enter a name to continue" — caption, `Theme.Colour.accentSoft`, appears below TextField only after a failed CTA tap.

**Completion action:** Call `AppStore.userCharacter = confirmed appearance`, `AppStore.displayName = displayName`, `CharacterPersistence.save(appearance)`, `PersistenceKeys.save(displayName)`, set `hasCompletedOnboarding = true`. Transition to `HomeView` with `.asymmetric` push (trailing insertion, no removal — the onboarding stack exits entirely).

**Progress dots:** Same treatment; 3rd dot active. Positioned above the CTA block.

### Screen 4 — Home Screen (`HomeView`)

**Layout:** ZStack — `IsometricRoomView` fills the full screen. Foreground VStack overlay anchored top and bottom.

**Top overlay:**
- Greeting "Hi, [displayName]." — Title font, `Theme.Colour.textPrimary`, left-aligned with `Theme.Spacing.lg` inset.
- Sub-greeting "Ready to lock in?" — Body font, `Theme.Colour.textSecondary`.
- Both on a subtle `LinearGradient` from `Theme.Colour.background.opacity(0.85)` to `.opacity(0)`, top-to-bottom.

**Avatar in room:** `AvatarView(appearance: appStore.userCharacter ?? .default, status: .idle, size: 80)` positioned at the desk slot in `IsometricRoomView`.

**Bottom action card:**
- `RoundedRectangle(cornerRadius: Theme.Radius.lg)` filled `Theme.Colour.surface`.
- Content: `AvatarView` (48×48 pt) + displayName (Headline) + status label "Idle" (Caption, textSecondary) — HStack.
- `Spacer()`
- "Start a room" button — amber pill (full-width within card, `Theme.Spacing.md` inset). Placeholder navigation for Phase 3.

## Reusable Component Contracts

### `AvatarView`

**Public signature:**
```swift
AvatarView(appearance: CharacterAppearance, status: AvatarStatus, size: CGFloat = 80)
```

**Layer order (ZStack, bottom to top):**
1. Base body shape (skin tone fill) — `AvatarBodyLayer`
2. Hair layer — `AvatarHairLayer` (style + colour)
3. Outfit layer — `AvatarOutfitLayer` (style + accentColour tint)
4. Status overlay — `AvatarStatusOverlay` (visible only when `status != .idle`)

**Status overlay visual contract (non-colour-only — MUST satisfy ONB-03 accessibility requirement):**

| AvatarStatus | Overlay ring colour | SF Symbol icon | Label text | Icon position |
|-------------|-------------------|----------------|------------|---------------|
| `.idle` | none | none | none | — |
| `.focused` | `Theme.Colour.accent` (amber) | `lock.fill` | "Focused" | bottom-right badge |
| `.deepFocus` | `#4DB8A4` (accentTeal) | `bolt.fill` | "Deep" | bottom-right badge |
| `.onBreak` | `Theme.Colour.textSecondary` | `cup.and.saucer.fill` | "Break" | bottom-right badge |
| `.distracted` | `Theme.Colour.forfeitRed` | `exclamationmark.triangle.fill` | "!" | bottom-right badge |
| `.finished` | `Theme.Colour.moneyGreen` | `checkmark.circle.fill` | "Done" | bottom-right badge |

Badge: 16×16 pt circle background (status ring colour at 90% opacity), icon at 10pt, positioned at `alignment: .bottomTrailing` offset `Theme.Spacing.xs` from the avatar frame edge.

**Explicitly both colour AND shape/icon for every non-idle state** — never colour alone. This satisfies WCAG non-colour-distinction for accessibility.

**All states built now, even though only `.idle` is used in Phase 2.** Phase 4 activates the others; API must not change.

**Reduce Motion:** Status overlay uses `.opacity` only transitions — no pulsing, no scale animation, regardless of Reduce Motion setting (overlays are informational, not decorative — never animate them in a way that conveys meaning through motion alone).

**Default size param:** 80 pt. Caller specifies: 120 in creator preview, 80 in payoff/home, 48 in Phase 3 roster (future), 40 in Phase 4 session HUD (future).

**D-21 vs D-21a:** The layer views (`AvatarBodyLayer`, `AvatarHairLayer`, `AvatarOutfitLayer`) are internal to `AvatarView`. Phase 2 builds them as code-drawn SwiftUI shapes (D-21a). If PNG layers are sourced in a Wave 3 enhancement, only these internal views change — the `AvatarView` public API is untouched. The planner MUST build D-21a first; do NOT stall Phase 2 on PNG sourcing.

## Animation & Motion Contract

### Master Reduce Motion Rule

Every animated view MUST read `@Environment(\.accessibilityReduceMotion) private var reduceMotion` and provide a calm fallback. Failure to implement this is a Phase 2 bug, not a polish item.

| Animation | Full motion | Reduce Motion fallback |
|-----------|------------|----------------------|
| Beat-to-beat transition | `.asymmetric(insertion: .move(.trailing) + .opacity, removal: .move(.leading) + .opacity)` | `.opacity` only; no slide |
| Beat 1 — concept lines stagger | Sequential offset+opacity entrance per line, 0.6s stagger | All lines visible simultaneously at `.opacity` 1 — no stagger, no offset |
| Beat 1 — supporting body + CTA fade-in | `.easeOut(duration: 0.4)` opacity | Immediate (`.animation(nil)`) |
| Beat 3 — avatar entrance (`PhaseAnimator`) | `.hidden → .riseUp (spring) → .settle (spring)` scale + offset + opacity | Single `.opacity` 0→1 fade, no scale, no offset |
| Beat 3 — heading text swap ("Almost there" → "Meet [name]") | `.transition(.opacity)` cross-fade 0.25s | `.transition(.opacity)` cross-fade 0.15s (shorter is fine; opacity-only is already calm) |
| Home screen — initial load | `AvatarView` fades in `.opacity` 0→1 over 0.3s | Same (opacity-only is already the Reduce Motion fallback) |
| Status overlay appearance | `.opacity` 0→1 only | Same (never use scale or slide on status overlays) |

### Timing constants (define as `private enum Timing` in `OnboardingView`)

| Constant | Value | Used by |
|----------|-------|---------|
| `beatTransitionSpring` | `.spring(duration: 0.4, bounce: 0.2)` | Beat-to-beat `withAnimation` |
| `conceptLineDelay` | `0.6` seconds between each line | `Task.sleep(for: .seconds(0.6))` in Beat 1 |
| `avatarEntranceRise` | `.spring(duration: 0.45, bounce: 0.5)` | PhaseAnimator `.riseUp` phase |
| `avatarEntranceSettle` | `.spring(duration: 0.3, bounce: 0.2)` | PhaseAnimator `.settle` phase |
| `headingSwapFade` | `.easeInOut(duration: 0.25)` | "Almost there" → "Meet [name]" transition |

## Accessibility Contract

| Rule | Implementation |
|------|---------------|
| Non-colour-only status cues (ONB-03) | Every `AvatarStatus` overlay uses colour + SF Symbol icon + text label — all three simultaneously |
| Reduce Motion respected | Every `withAnimation` call gates on `reduceMotion`; every `PhaseAnimator` has a static fallback |
| Touch targets ≥ 44×44 pt | All ◀▶ stepper buttons: `.frame(width: 44, height: 44)` minimum |
| Text contrast | `Theme.Colour.textPrimary` (`#F2E9D9`) on `Theme.Colour.background` (`#1E190F`) — warm cream on deep brown, WCAG AA compliant by visual inspection (ratio ≥ 7:1) |
| TextField keyboard handling | Set `.submitLabel(.done)` on display-name TextField; dismiss keyboard on submit |
| VoiceOver labels | `AvatarView` exposes `.accessibilityLabel("Avatar: [appearance.description], status: [status.label]")` — implement `description` computed property on `CharacterAppearance` |
| Avatar status badge | `.accessibilityHidden(true)` on badge image/colour; the parent `AvatarView` accessibility label already carries the status text |
| Skip button | `.accessibilityLabel("Skip onboarding")` — explicit label, not just "Skip" |
